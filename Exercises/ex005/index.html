Monospaced Writing : Source Code

Source code is the text-based set of instructions that programmers write to create software and computer programs.

Monospaced writing is a type of text where each character takes up the same amount of horizontal space. In monospaced fonts, like Courier or Consolas, an "i" takes up as much space as a "w," making it easy to align text neatly in rows and columns.

Monospace is good for source code because it ensures that each character occupies the same amount of space, making code more readable and organized, especially when aligning text and code elements in columns and rows.

Example:

function calculateFactorial(n) {
if (n === 0 || n === 1) return 1;
return n * calculateFactorial(n - 1); }

Is harder to read than if we use CODE to envelope the code section - makeing it monospaced writing instead.

function calculateFactorial(n) {
if (n === 0 || n === 1) return 1;
return n * calculateFactorial(n - 1); }

Another thing is PRE - Keeps text formatting, useful for displaying code blocks. While CODE Styles (monospace) inline code within a sentence.

Example (instead of BR we use PRE): 

                
function calculateFactorial(n) {
if (n === 0 || n === 1) return 1;
return n * calculateFactorial(n - 1); }

Please note that with PRE every space will be counted in the website.

Complete Citations

Simple Quotes (Q): Used for short inline quotations within a sentence.

Block Quotes (BLOCKQUOTE): Used for longer quotations that are displayed as separate blocks of text, often indented.

Example:

Lorem ipsum dolor sit amet, consectetur adipisicing elit. Explicabo rerum corporis, in aut tenetur similique dolorem nam aperiam consectetur assumenda repudiandae reiciendis error rem est numquam iste alias? Sed, odit?

To reference the source of the citation, simply write inside of the opening tag for blockquote "cyte=" and paste the source link.
But keep in mind this link will only be visible by search mechanisms like Google. It is good use for giving rights to respective authors.
